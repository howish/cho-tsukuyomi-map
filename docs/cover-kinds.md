# Cover image classification — お品書き / cover / announce

`cover_urls[].kind` の分類基準。**画像そのものを見て判定する** — 投稿テキストは
proxy にすぎない（「お品書き」と書いてなくてもお品書き画像は山ほどある。
2026-06-10 ヤオヨロー見直しで判明: テキスト判定だと 34 booth 中 6 件を
誤分類した）。

## kind = "oshinagaki" (お品書き)

**頒布物を列挙したデザイン済みリスティング画像。** 判定基準（画像内に
見えるもの）:

- 頒布物（本/グッズ）が**価格つき**で並んでいる
- 典型的にはイベント名・開催日・スペース番号・サークル名入り
- 新刊/既刊の区分、サムネイル表紙のグリッド配置が多い
- **1 冊だけの「新刊カード」もお品書き** — 単品でもスペック
  （判型/ページ数/年齢区分）+ 価格 + スペース番号がレイアウトされていれば
  リスティングとして機能している（例: ヤオ-16 製麺所、ヤオ-20 EXTRA SYSTEM.）

テキストに「お品書き/おしながき/品書」が無くても上記を満たせば oshinagaki。
逆に「お品書きは後ほど」のような**予告だけのポスト**は違う。

## kind = "cover" (表紙)

本の表紙アートそのもの、またはタイトルページ。1 枚絵 + タイトル文字。
価格グリッドは無い。印刷見本写真（刷り上がった実物の写真）も cover 扱い。

## kind = "announce" (告知)

参加告知ビジュアルで、価格リスティングでも表紙でもないもの
（例: 「参加します」バナー、キービジュアル流用）。

## untagged のまま

- サンプルページ（漫画本文ページ）
- 展示物・グッズの実物写真（例: ヤオ-35 のぬいぐるみ）
- どれにも当てはまらない画像

## カバーにしてはいけないもの（extract が tier -1 で排除）

完売報告 / 撤収・設営写真 / 戦利品（他人の本！誤帰属事故）/ 差し入れ /
通販受付 / ボツ案。

## クロスイベント検査

画像内のスペース番号がこのイベントの体系と違う（例: yaoyoro なのに
`A-04` や `る54a`）→ **別イベントの素材**。使わない。

## 運用

- 自動抽出 (`scripts/ops/extract_cover_urls.py`) はテキスト keyword で
  仮タグするだけ。**目視（または vision agent）で画像確認したら
  `kind` を正しく直し `manual: true` でピン**する — refresh で消えない。
- 分類ジョブを agent に出すときはこのファイルを prompt に含めること。
